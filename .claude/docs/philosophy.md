# Philosophy — Tiny Studio

This template exists because **one great collaboration loop** beats **forty
org-chart agents**.

## What we stole from “big studio” templates

- **Clear roles** — so the model knows which hat to wear.
- **Slash workflows** — so you don’t re-explain process every session.
- **Honest QA** — fun dies in silence; someone must say the uncomfortable thing.

## What we refused

- **Agent armies** — noise scales faster than quality.
- **Escalation trees** — you are not a ticket system.
- **Documentation theater** — paper is not progress.

## How decisions happen

1. Agents **surface options and tradeoffs**.
2. Agents may **disagree**; you see the tension.
3. **You choose.** The repo stays yours.

## Taste

We optimize for games that are **readable**, **expressive**, **intentionally
scoped**, and **possible to finish** alone or in a tiny group. Retro lessons
matter: clarity, rhythm, and memorable simplicity. Modern tools matter when they
**remove friction**, not when they **add complexity**.
