# /art-direction — Make it coherent

## Lead

**game-artist** drives. Consult **game-designer** when clarity affects mechanics;
**game-developer** when tech limits shaders, UI stack, or performance.

## Steps

1. **Read** `design/pillars.md` and any feature specs that matter.
2. **Audit quickly**: list 3 strengths + 3 **honest** problems in current visuals
   (from files, screenshots user provides, or described state).
3. **Define**:
   - **Palette** — primary, secondary, accent, “danger/success” (hex or names)
   - **Shape language** — curves vs hard edges, proportions
   - **Motion** — timing personality (snappy vs floaty), restraint level
   - **UI rules** — hierarchy, one example screen described in words
   - **FX restraint** — particle budget vibe (minimal / playful / chaotic)
4. **Cohesion rules** — 5–10 bullet **do / don’t** pairs.
5. **Write** or update `design/art-notes.md` (create if absent) as the **source
   of truth** for look — keep it **two pages max** unless asked.

## Tone

Encouraging but **specific**. Replace “make it pop” with **contrast, scale, or
timing** guidance.

## Do not

- Demand a certain engine or pipeline
- Add dozens of reference images inside the repo without user ask
